# Story Mode 문법 해설 기능 구현 계획

동화 모드(Story Mode)에서 각 문장의 세부적인 문법과 단어 뜻, 핵심 요약을 보여주는 기능을 추가하기 위한 구현 계획입니다.

## User Review Required

UI/UX 디자인과 JSON 데이터 구조에 대해 확인하고 선택해 주셔야 합니다. 아래의 아이디어를 확인하시고 피드백을 부탁드립니다.

---

## 1. UI/UX 아이디어 제안

학습의 흐름을 끊지 않으면서도 미적(Zen 스타일) 감각을 유지하기 위한 두 가지 UI 옵션을 제안합니다.

### Option A: Glassmorphism 모달 (추천)
- **작동 방식**: 문장 아래에 작고 예쁜 `💡 문법 해설` 버튼을 배치합니다. 이 버튼을 누르면 화면 중앙에 반투명한 블러 효과(Glassmorphism)가 적용된 모달 창이 부드럽게 나타납니다.
- **장점**: 
  - 현재 타이핑 화면을 복잡하게 만들지 않고 깔끔하게 유지합니다.
  - 설명이 길어져도 스크롤을 넣어 깔끔하게 보여줄 수 있습니다.
  - 기존의 `finish-dialog`와 디자인 일관성을 유지할 수 있습니다.
- **디자인 포인트**: 
  - 항목별(주어, 동사 등)로 태그 스타일의 뱃지를 달아 가독성을 높입니다.
  - '핵심 요약'은 강조된 박스(Callout) 형태로 하단에 배치합니다.

### Option B: 인라인 아코디언 (Expandable Card)
- **작동 방식**: 문장 아래에 `🔽 문법 해설 보기` 버튼을 누르면, 카드 UI 아래쪽이 부드럽게 확장되면서 그 자리에 바로 설명이 나타납니다.
- **장점**: 
  - 모달처럼 시야를 가리지 않고, 타이핑 창을 보면서 설명을 읽을 수 있습니다.
- **디자인 포인트**: 
  - CSS `max-height` 트랜지션을 이용해 부드럽게 스르륵 열리는 애니메이션을 적용합니다.

> **질문 1:** Option A(모달 창)와 Option B(인라인 확장) 중 어떤 방식이 더 마음에 드시나요?

## 2. JSON 데이터 구조 변경안

`kaninotokoya.JSON`의 각 문장 객체에 `grammar` 필드를 추가하여 데이터를 구조화합니다.

```json
{
    "id": 1,
    "kanji": "蟹がいろいろ考えたあげく、とこやをはじめました。",
    "kana": "かにがいろいろかんがえたあげく、とこやをはじめました。",
    "romaji": "kani ga iroiro kanngaeta ageku tokoya o hajimemashita",
    "meaning": "게가 여러 가지를 생각한 끝에, 이발소를 시작했습니다.",
    "grammar": {
        "breakdown": [
            {
                "phrase": "蟹(かに)が",
                "role": "주어",
                "points": [
                    "蟹(かに): 게 (주인공)",
                    "が: 주격 조사 (~이/가)"
                ]
            },
            {
                "phrase": "いろいろ 考えた(かんがえた)",
                "role": "부사 + 동사",
                "points": [
                    "いろいろ: 여러모로, 다각도로 (부사)",
                    "考えた: '생각하다(考える)'의 과거형입니다."
                ]
            },
            {
                "phrase": "あげく",
                "role": "문법 포인트: ~한 끝에",
                "points": [
                    "구조: 동사의 과거형(た형) + あげく",
                    "뜻: '~한 끝에', '결국 ~하다'",
                    "설명: 어떤 일을 오랫동안 지속하거나 여러 번 반복한 뒤에 어떤 결과에 도달했을 때 사용합니다."
                ]
            },
            {
                "phrase": "とこやを",
                "role": "목적어",
                "points": [
                    "とこや: 이발소 / 이발사",
                    "を: 목적격 조사 (~을/를)"
                ]
            },
            {
                "phrase": "はじめました",
                "role": "동사의 정중한 과거형",
                "points": [
                    "はじめる: 시작하다",
                    "~ました: '~했습니다' (정중한 과거형)"
                ]
            }
        ],
        "core_summary": "이 문장의 주인공인 게가 이발소를 차리기까지 꽤 깊은 고민의 시간(あげく)이 있었음을 알 수 있는 문장입니다."
    }
}
```

> **질문 2:** 위와 같은 데이터 구조가 사용자가 원하는 방식과 일치하나요? (원하시는 경우 HTML로 렌더링하기 편하도록 구조를 조정할 수 있습니다.)

## 3. 구현 단계 (Proposed Changes)

1. **데이터 업데이트**: `kaninotokoya.JSON`의 첫 번째 문장에 위 샘플 데이터를 적용.
2. **HTML/CSS 추가 (`index.html`, `index.css`)**:
   - `sentence-view` 내부에 `💡 문법 해설` 버튼 추가.
   - 선택된 UI 옵션(모달 또는 아코디언) 마크업 및 스타일링 추가.
   - 항목(`phrase`, `role`, `points`)을 시각적으로 예쁘게 렌더링하는 CSS 클래스 작성.
3. **JavaScript 로직 수정 (`index.html`)**:
   - 문장 로드 시 현재 문장에 `grammar` 데이터가 있는지 확인하고, 있으면 버튼을 표시.
   - 버튼 클릭 시 `grammar` 데이터를 파싱하여 모달/아코디언 안에 동적으로 HTML 요소를 생성하고 표시하는 함수(`showGrammar()`) 추가.
