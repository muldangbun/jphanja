# Zen Kanji Master - 프로젝트 상태 보고서 (2026-05-01)

본 문서는 프로젝트의 현재 개발 상태를 상세히 기록하여, AI Antigravity가 작업 상황을 즉시 파악하고 보조할 수 있도록 돕기 위해 작성되었습니다.

## 1. 프로젝트 개요
- **목표**: 1학년부터 6학년까지의 한자 및 필수 어휘, 그리고 문장 단위의 동화 읽기를 Zen 스타일의 미니멀한 UI에서 학습하는 웹 애플리케이션.
- **포커스**: 집중(Zen), 시각적 미학(Glassmorphism), 부드러운 반응성 및 즉각적인 피드백.

## 2. 주요 학습 모드
- **한자 모드 (Kanji Mode)**: 
  - 선택한 학년의 한자를 하나씩 학습.
  - **한국식 훈음(뜻과 소리) 추가**: 1학년(80자) 및 2학년(160자) 전체 한자에 대해 한국식 훈음 데이터를 추가하여 학습 편의성 강화 (예: 羽 -> 날개 우).
  - 음독(Onyomi) -> 훈독(Kunyomi) 순서로 진행.
- **단어 모드 (Word Mode)**: 
  - 한자와 관련된 주요 예시 단어들을 순차적으로 학습.
- **동화 읽기 모드 (Story Mode)**: 
  - *'게의 장사(かにの商売)'* 수록 (총 38문장).
  - 문장 단위 타이핑 연습 및 상세 문법 해설(Grammar Explanation) 기능.
  - `ruby` 태그를 활용한 동적 후리가나 표시 및 타이핑 진척도에 따른 한자 노출 기능.

## 3. 핵심 기술 및 시스템
- **데이터 무결성 시스템**:
  - `kaninotokoya.JSON` 데이터 복구 및 정제 완료: 깨진 JSON 구조를 복구하고 문장 내의 모든 한국어 잔재를 제거하여 순수 일본어 데이터로 정규화.
  - **로마자 표기 통일**: 모든 `ん` 입력을 `nn`으로 통일하여 입력 일관성 확보.
- **Romaji-to-Kana 엔진**:
  - `romajiToKana` 함수를 통한 실시간 변환.
  - `nn` -> `ん` 처리 고도화 및 `っ`(작은 つ) 자동 처리 로직 포함.
- **타이핑 피드백 시스템 개선**:
  - **로마자 가이드 낱자 단위 하이라이트**: 기존의 가나(한 글자) 단위가 아닌, 사용자가 입력한 알파벳 하나하나에 즉각적으로 반응하도록 하이라이트 로직 개선.
  - `normalizeText` 함수: 조사(`は`->`わ` 등) 및 요음 처리를 정규화하여 사용자 편의성 제공.
- **버전 관리 및 호스팅**:
  - **Git/GitHub 연동**: 로컬 저장소를 초기화하고 [muldangbun/jphanja](https://github.com/muldangbun/jphanja.git)에 전체 소스 코드 푸시 완료.
  - **GitHub Pages**: 웹 호스팅 설정 준비 완료.

## 4. UI/UX 및 코드 품질
- **디자인 시스템**: Glassmorphism 기반의 카드 레이아웃 및 애니메이션 효과.
- **훈음 힌트 시각화**: 한자 학습 모드에서 한국식 훈음을 강조된 보조 텍스트(`.kor-huneum`)로 표시하도록 CSS 및 JS 로직 업데이트.
- **브라우저 호환성**: CSS 표준 속성 준수 및 IDE 경고 제거.

## 5. 프로젝트 구조
- `index.html`: 메인 로직, UI 구조, 상태 관리.
- `index.css`: 디자인 토큰 및 레이아웃 정의 (한국어 훈음 스타일 포함).
- `kanji_grade1.json` / `kanji_grade2.json`: 각 학년별 한자/단어 데이터 (한국식 훈음 데이터 포함).
- `kaninotokoya.JSON`: 정제된 '게의 장사' 동화 및 상세 문법 데이터 (Source of Truth).

## 6. 개발자 참고 사항 (AI 전용)
- **한자 데이터 추가 규칙**: 신규 학년 데이터 추가 시 `kor_meaning`과 `kor_sound` 필드를 반드시 포함할 것.
- **로마자 하이라이트**: `state.rawInput`을 목표 로마자와 실시간 비교하여 하이라이트 위치를 결정함.
