# Zen Kanji Master - 프로젝트 상태 보고서 (2026-05-23)

본 문서는 프로젝트의 현재 개발 상태를 상세히 기록하여, AI Antigravity가 작업 상황을 즉시 파악하고 보조할 수 있도록 돕기 위해 작성되었습니다.

## 1. 프로젝트 개요
- **목표**: 1학년부터 6학년까지의 한자 및 필수 어휘, 그리고 문장 단위의 동화 읽기를 Zen 스타일의 미니멀한 UI에서 학습하는 웹 애플리케이션.
- **포커스**: 집중(Zen), 시각적 미학(Glassmorphism), 부드러운 반응성 및 즉각적인 피드백.

## 2. 주요 학습 모드
- **한자 모드 (Kanji Mode)**: 
  - 선택한 학년의 한자를 하나씩 학습.
  - **한국식 훈음(뜻과 소리) 추가**: 1학년(80자) 및 2학년(160자) 전체 한자에 대해 한국식 훈음 데이터를 추가하여 학습 편의성 강화 (예: 羽 -> 날개 우).
  - 음독(Onyomi) -> 훈독(Kunyomi) 순서로 진행.
- **단어 모드 (Word Mode)**: 
  - 한자와 관련된 주요 예시 단어들을 순차적으로 학습.
- **상황별 회화 모드 (Conversation Mode)**: 
  - 편의점 등 실생활 상황별 회화 시나리오 및 문법 학습 기능 추가 (`convenience_scenesv1.JSON`, N3 레벨).
- **동화 읽기 모드 (Story Mode)**: 
  - *'게의 장사(かにの商売)'* 수록 (총 38문장, 오디오: 즈다몬 캐릭터).
  - *'나비와 애벌레(なびわぷる)'* 수록 (총 8문장, 오디오: 後鬼 인간 버전 - 후키).
  - *'달팽이의 슬픔(でんでんむしのかなしみ)'* 수록 (총 16문장).
  - 문장 단위 타이핑 연습 및 상세 문법 해설(Grammar Explanation) 기능.
  - `ruby` 태그를 활용한 동적 후리가나 표시 및 타이핑 진척도에 따른 한자 노출 기능.

## 3. 핵심 기술 및 시스템
- **데이터 무결성 시스템**:
  - `kaninotokoya.JSON` 데이터 복구 및 정제 완료.
  - `nabiwapull.JSON` 신규 추가: 전각 공백 제거, 괄호 형식 후리가나(`草花（くさばな）`) → 한자 단독 표기(`草花`)로 정제 완료. 후리가나는 `alignFurigana()` 함수가 `kana` 필드를 이용하여 자동 처리.
  - **로마자 표기 통일**: 모든 `ん` 입력을 `nn`으로 통일하여 입력 일관성 확보.
- **동화 오디오 경로 구조 개선 (2026-05-21)**:
  - 기존 하드코딩된 경로 `audio/story/` → 동화 이름 기반 동적 경로 `audio/${storyName}/`로 개선.
  - `state.currentStoryName`에 선택한 동화 파일명(확장자 제거)을 저장하고, `loadAudioBuffer()`에서 동적으로 경로 조합.
  - `audio/story` 폴더 → `audio/kaninotokoya`로 이름 변경 (다작품 혼용 방지).
  - `audio/nabiwapull/` 신규 폴더 생성 및 MP3 파일 추가 (0.mp3 ~ 7.mp3).
- **Romaji-to-Kana 엔진**:
  - `romajiToKana` 함수를 통한 실시간 변환.
  - `nn` -> `ん` 처리 고도화 및 `っ`(작은 つ) 자동 처리 로직 포함.
- **타이핑 피드백 시스템**:
  - **로마자 가이드 낱자 단위 하이라이트**: 사용자가 입력한 알파벳 하나하나에 즉각적으로 반응.
  - `normalizeText` 함수: 조사(`は`->`わ` 등) 및 요음 처리를 정규화.
- **동화 모드 오디오북 자동화**:
  - 오디오북 모드 활성화 시 문장 재생 후 2초 간격으로 다음 문장 자동 진행.
  - 오디오북 모드 활성화 시 원문 글씨 검은색(#333) 고정으로 가독성 강화.
- **버전 관리 및 호스팅**:
  - **Git/GitHub 연동**: [muldangbun/jphanja](https://github.com/muldangbun/jphanja.git) 저장소에 최신 코드 및 데이터 동기화 완료.

## 4. UI/UX 및 모바일 최적화
- **디자인 시스템**: Glassmorphism 기반의 카드 레이아웃 및 애니메이션 효과.
- **음독/훈독 상호작용 토글**: 한자 학습 시 상단의 상태 태그('음독'/'훈독')를 클릭하여 모드를 상호 전환할 수 있는 기능 추가. (반대 발음이 없는 경우 error-shake 애니메이션 피드백 제공).
- **훈음 힌트 시각화**: 한자 학습 모드에서 한국식 훈음을 강조된 보조 텍스트(`.kor-huneum`)로 표시.
- **가상 자판(Virtual Keyboard) 지능형 노출 제어**:
  - 오디오북 모드 활성화 시 화면 공간 확보를 위해 가상 자판을 자동으로 숨김 처리.
  - CSS 클래스(`.audiobook-active`) 기반 제어로 UI 전환의 안정성과 성능 최적화.
- **모바일/사파리 레이아웃 최적화**:
  - `100dvh` 적용 및 `flex-start` 정렬을 통해 콘텐츠 잘림 방지.
  - 사파리 브라우저의 flex 가로 너비 버그 해결 (`width: 100%`).
  - 아이폰 텍스트 그라데이션 렌더링 버그 해결 (`line-height`, `padding` 조절).

## 6. 개발자 참고 사항 (AI 전용)
- **한자 데이터 추가 규칙**: 신규 학년 데이터 추가 시 `kor_meaning`과 `kor_sound` 필드를 반드시 포함할 것.
- **동화 JSON 데이터 규칙**:
  - `kanji` 필드: 전각 공백 없이, 한자 그대로 (괄호 후리가나 사용 금지). `alignFurigana()`가 자동 처리.
  - `kana` 필드: 전각 공백 없이, 전체 히라가나 발음 기재.
  - `grammar` 필드: 각 항목마다 `points` 배열과 `summary` 문자열 포함.
- **오디오 경로 규칙**: 동화 추가 시 `audio/{동화파일명(확장자제외}/` 폴더에 `0.mp3`부터 순서대로 배치. 코드에서 `state.currentStoryName`으로 자동 매핑됨.
- **동화 버튼 추가**: `index.html`의 `<div class="grade-grid">` 내부에 `selectStory('파일명.JSON')` 형식으로 버튼 추가.
- **로마자 하이라이트**: `state.rawInput`을 목표 로마자와 실시간 비교하여 하이라이트 위치를 결정함.
- **코드 구현 원칙**: 사용자의 명시적인 승인("수정해 줘", "진행해" 등)을 받은 후에만 파일 수정을 진행할 것.
