# Zen Kanji Master - 프로젝트 상태 보고서 (2026-04-29)

본 문서는 프로젝트의 현재 개발 상태를 상세히 기록하여, AI Antigravity가 작업 상황을 즉시 파악하고 보조할 수 있도록 돕기 위해 작성되었습니다.

## 1. 프로젝트 개요
- **목표**: 1학년부터 6학년까지의 한자 및 필수 어휘, 그리고 문장 단위의 동화 읽기를 Zen 스타일의 미니멀한 UI에서 학습하는 웹 애플리케이션.
- **포커스**: 집중(Zen), 시각적 미학(Glassmorphism), 부드러운 반응성 및 즉각적인 피드백.

## 2. 주요 학습 모드
- **한자 모드 (Kanji Mode)**: 
  - 선택한 학년의 한자를 하나씩 학습.
  - 음독(Onyomi) -> 훈독(Kunyomi) 순서로 진행.
- **단어 모드 (Word Mode)**: 
  - 한자와 관련된 주요 예시 단어들을 순차적으로 학습.
- **동화 읽기 모드 (Story Mode)**: 
  - *'게의 장사(かにの商売)'* 수록 (총 38문장).
  - 문장 단위 타이핑 연습 및 상세 문법 해설(Grammar Explanation) 기능.
  - `ruby` 태그를 활용한 동적 후리가나 표시 및 타이핑 진척도에 따른 한자 노출 기능.

## 3. 핵심 기술 및 시스템
- **데이터 무결성 시스템 (신규)**:
  - `kaninotokoya.JSON` 데이터 정제 완료: 모든 문장의 한글 혼입 오류 제거 및 일본어 텍스트 정규화.
  - **로마자 표기 통일**: 모든 `ん` 입력을 `nn`으로 통일하여 입력 일관성 확보.
- **Romaji-to-Kana 엔진**:
  - `romajiToKana` 함수를 통한 실시간 변환.
  - `nn` -> `ん` 처리 고도화 및 `っ`(작은 つ) 자동 처리 로직 포함.
- **타이핑 피드백 및 정규화**:
  - `normalizeText` 함수: 조사(`は`->`わ` 등) 및 요음 처리를 정규화하여 사용자 편의성 제공 (한/일 문자 혼용 버그 수정 완료).
  - `charKanaMapping`: 글자 단위 정밀 매핑을 통한 진척도 시각화.
- **TTS (Text-to-Speech)**:
  - 브라우저 Web Speech API 사용. 자연스러운 일본어 발음 제공.

## 4. UI/UX 및 코드 품질
- **디자인 시스템**: Glassmorphism 기반의 카드 레이아웃 및 애니메이션 효과.
- **코드 최적화**: 
  - CSS 표준 속성(`appearance`, `background-clip`) 적용으로 브라우저 호환성 강화 및 IDE 경고 제거.
  - 미정의 CSS 변수(`var(--accent)` 등) 정리.
- **문법 해설 (Option A - Glassmorphism 모달)**: 각 문장별 상세 단어 뜻, 문법 포인트, 문장 요약을 제공하는 반투명 Glassmorphism 기반 전용 모달 시스템 구현 완료.

## 5. 프로젝트 구조
- `index.html`: 메인 로직, UI 구조, 상태 관리, 인라인 스타일 보조.
- `index.css`: 디자인 토큰 및 레이아웃 정의 (가독성 및 표준 준수 업데이트).
- `kanji_grade1.json`: 1학년 한자/단어 데이터.
- `kaninotokoya.JSON`: 정제된 '게의 장사' 동화 및 상세 문법 데이터 (Source of Truth).

## 6. 개발자 참고 사항 (AI 전용)
- **로마자 통일 규칙**: `ん`은 반드시 `nn`으로 표기하며, 데이터 수정 시 이 규칙을 엄수할 것.
- **입력 안전 장치**: `typing-input` 처리 시 `lastChar` 존재 여부 확인 등 예외 처리가 반영되어 있음.
- **문법 데이터**: `grammar` 오브젝트 내 `points` 배열을 통해 모달 콘텐츠가 생성됨.
